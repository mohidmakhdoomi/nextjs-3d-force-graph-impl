### 3. Hypothesis Assessment

**H1 (stray node capture)** is the strongest hypothesis and the spec correctly identifies it as such. The reasoning chain is sound:

1. DragControls raycasts all node meshes at pointerdown
2. If a node is under `(150, 450)`, `dragstart` fires → `controls.enabled = false` → TrackballControls' `update()` is skipped entirely → camera delta ≈ 0
3. Firefox-specific zoom difference means closer camera → larger node projections → higher probability of node-under-point
4. Random force layout seed per page load explains per-run nondeterminism
5. Passing when repeated alone is consistent (same seed tends to reproduce same layout)

The predicted discriminators (`fixedNodeCount 0→1`, `controls.enabled === false`, `dragstart` event) are clean and unambiguous — they'd prove H1 from a single occurrence.

The spec's proposed fix for H1 (verified-background start point via an inverse `pickNodeScreenPoint`) is the right shape — it turns a probabilistic premise into a verified one, which is strictly stronger.
